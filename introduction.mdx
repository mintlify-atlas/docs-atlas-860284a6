VES gives small teams and AI Agents a simple, high-performance pipeline for collecting, processing, and forwarding observability data — without the operational overhead of bloated enterprise tooling.

The platform centers on the **Core Agent**: a single Rust binary you deploy alongside your workloads to tail log files, read from systemd journals, and accept data over TCP/UDP/Unix sockets — then forward enriched events to the **HEIMDELL Server**.

## Why VES?

Most observability stacks are designed for large engineering organisations. They require dedicated infrastructure teams to operate, consume significant resources, and charge accordingly.

VES is designed around a different set of assumptions:

- **Small teams move fast** — the platform stays out of your way and keeps configuration simple.
- **Reliability matters** — the Core Agent uses an LMDB-backed recovery layer so events survive restarts and crashes.
- **Rust performance** — zero garbage collection pauses, minimal memory footprint, safe concurrency.
- **Open source** — AGPL-3.0 licensed. You can read every line, contribute improvements, and self-host forever.

## Key components

| Component | Description |
|-----------|-------------|
| **core-agent** | Deployed on each host. Collects logs from files, journald, and sockets; forwards to HEIMDELL Server. |
| **heimdell** | The server-side component that receives events from one or more Core Agents. *(In development)* |
| **ves_lib** | Shared Rust library providing Protobuf-generated data contracts used by both binaries. |
